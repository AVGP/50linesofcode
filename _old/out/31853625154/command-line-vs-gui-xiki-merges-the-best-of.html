Command line vs. GUI - Xiki merges the best of both worlds

September 19th 2012

#Command line vs. GUI - Xiki merges the best of both worlds

I often hear people say "The console is so much faster and much better than any graphical interface in the world!" and as frequently as that I hear people say "You can't do that on the command line, this graphical interface is much easier and more comfortable!".

Personally, I switch a lot between graphical interfaces and the command line. Why? Because I don't evangelize about it - I need to get a job done. Sometimes getting something done is faster on the console, sometimes it is easier to use a GUI.

Here is an example: I am perfectly fine with SSHing to a server and use vim there to edit some configuration files, updating the system, searching logs, debugging things, etc.

But on the other hand IDEs such as Eclipse, Netbeans, Springsource are more comfortable for a lot of other tasks- they offer me a huge number of tools comfortably at my fingertips. Refactoring? Easy. Rename a whole package and any references to it? Easy!

Doing that with the command line is not impossible, but it is a little more tricky than clicking a button.


Here comes Xiki!

Xiki is a project to merge best of both worlds. You can use it from your favourite text editor - supported at the moment are Emacs, Vim, Textmate and CodeMirror. Others may be added through extensions.

Installing is easy, as long as you have Ruby on your system - you can install it using Gem:

$ gem install xiki

or you can get it from GitHub. 

It allows you to do awesome things - you can run command line commands, expand and fold the output, run ruby code,work with databases, connect to remote systems, try HTML and CSS in your browser, create a rails project and many many more.
