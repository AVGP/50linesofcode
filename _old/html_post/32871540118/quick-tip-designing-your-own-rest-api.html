Quick tip: Designing your own REST API

2012-10-04 13:59:24 GMT

If you want to set up a REST API for your project, you will quickly see a lot of questions arise, especially if you’ve been working with a few different APIs yourself.

  • Should URLs use plural or singular?
  • How to deal with clients that can’t do real REST (e.g. IE 8 Javascript has issues with cross-origin HTTP-PUT and DELETE)?
  • How should URLs look like? Should they have verbs inside?
  • How to version the API?
  • And a lot more.

The answer to some of these questions is: Whatever you like better.

But I have been looking around for some advice from people who are focused on API stuff day in day out.

Now I want to share my findings with you:

Apigee

Apigee is a company that helps you run your API, so you don’t have to deal with performance, rate limiting, etc.

They have quite a high-line profile of customers: AT&T, ComCast, Gamespy, Getty Images, ebay, Dell, Swift, Moneybookers and a bunch of other well-known names.

The interesting thing: They have a blog and a free e-book about API Design.

The Blog posts have been collected and put together as a free e-book and they have some more available for purchase or download.

API Craft Google Group

The other useful thing I stumbled upon is the API Craft Group where a lot of people actively discuss API design questions, do’s&don'ts and it is really insightful to read the discussions.
